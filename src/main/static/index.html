Create efficient Reactive systems

Reactor is a foundational library for building non-blocking applications on
the JVM based on the Reactive Streams Specification

Core Library

2.1.0.M1

A fully non-blocking foundation with efficient demand management. Its reactive flow-control implementation leaves not much risk for asynchronous publishing badly overflowing.

Non blocking IO

2.1.0.M1

Suited for "Microservices" Architecture, Reactor IO offers Backpressure-Ready IO components for HTTP (including websockets), TCP and UDP Clients/Servers/Bus. Reactive Encoding/Decoding is fully supported.

Stream & Bus

2.1.0.M1

Reactor Framework expose ready-to-consume composable Stream and Eventing API. "Stream" is a contract derived from Reactive Extensions while EventBus offers a flexible routing model.

Turbo Message Passing

Reactor Processors, Operators and Timers can sustain high throughput rates on the order of 10's of millions of messages per second. Plus its low memory footprint should go under most of the radars. Since they are mostly lock-free, contention and cascading deadlocks are stay under control.

Friendly Adoption

As a Reactive Engine/SPI, both reactor Core and IO modules expose reactive streams constructs for focused use cases, eventually combined with Spring, RxJava, Akka Streams, Ratpack... As a Reactive API, reactor framework modules will offer rich consumable features like composition and eventing.

Guardian of the Latency

Scaling-Out to overcome latency and slow microservices is an important tool. Cloud Native apps will give a chance at addressing the issue with asynchronous architectures. To assist such design, Reactor offers non-blocking and backpressure-ready IO solutions including Unicast/Multicast messaging or TCP/HTTP/UDP client & servers.
