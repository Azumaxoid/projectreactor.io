What is Reactor?

Reactor is a foundational library for building reactive fast-data applications on the JVM. You can use any JVM langauge, from Java 7 or Java 8, to Scala, Groovy, or even Nashorn JavaScript. It’s designed to be fast and efficient to enable you to write asynchronous applications that can manage extremely high workloads with very little overhead. It builds on the extreme throughput capabilities of the Disruptor RingBuffer to achieve throughput in the 10’s of millions of operations per second (even up to 100’s if you have enough hardware horsepower).[1]

Reactive Streams

The Reactor team are core contributors to the Reactive Streams Specification. Reactive Streams is "an initiative to provide a standard for asynchronous stream processing with non-blocking back pressure on the JVM."

There are certain characteristics of a system that "speaks" Reactive Streams beyond just using a common, shared API. The components inside Reactor’s Stream actions behave in a predictable way since they implement the Reactive Streams standard. This is a type of inter-operability among implementations that goes far beyond a common set of interfaces. In fact, Reactive Streams implementations can largely be depended upon to Just Work™.

Getting Started

To add Reactor to an existing project, you just need to include the appropriate artifact. The basic components of Reactor like EventBus and Stream are in the reactor-core artifact. Additional functionality is in the reactor-net, reactor-groovy, and reactor-logback modules.

build.gradle
ext {
    reactorVersion = '2.1.0.BUILD-SNAPSHOT'
    }

    repositories {
    // maven { url 'http://repo.spring.io/libs-release' } (1)
    // maven { url 'http://repo.spring.io/libs-milestone' } (2)
    maven { url 'http://repo.spring.io/libs-snapshot' } (3)
    }

    dependencies {
    //Core stuff (Processors, Functional interfaces...)
    compile "io.projectreactor:reactor-core:$reactorVersion"

    //Everything Stream
    compile "io.projectreactor:reactor-stream:$reactorVersion"

    //Everything Routing (Event Bus)
    //compile "io.projectreactor:reactor-bus:$reactorVersion"

    //Everything Net
    //compile "io.projectreactor:reactor-net:$reactorVersion"
    //compile "io.netty:netty-all:4.0.31.final"
    }
1 Uncomment to use a release version.
2 Uncomment to use a milestone version.
3 Uncomment to use a snapshot version.

Now that you’ve added the Reactor library to your project, you can start using the Stream class to perform high-volume, low-latency, asynchronous work. It’s really almost deceptively simple. Here’s a complete Hello World to get you started.

ReactorHelloWorld.java
import reactor.Processors;
    import org.reactivestreams.Processor;
    import reactor.rx.Streams;

    public class ReactorHelloWorld {
    public static void
    main(String... args) throws InterruptedException {
    Processor<String, String> sink =
    Processors.topic(); (1)

    Streams.create(sink) (2)
    .map(String::toUpperCase) (3)
    .consume(s -> System.out.printf("s=%s%n", s)); (4)

    sink.onNext("Hello World!"); (5)

    sink.onComplete(); (6)
    }
    }
1 Create an asynchronous message passing processor, signals coming in onXXX will be dispatched asynchronously.
2 Decorate the Processor with the Reactor Stream API.
3 Transform input to upper-case and implicitly broadcast downstream.
4 Consume the transformed input and print to STDOUT. This will start the subscriber thread attached to our sink.
5 Publish a value into the Processor.
6 Release Processor resources (non daemon thread) allowing for JVM exit.
1. Reactor microbenchmarks are maintained in a separate GitHub repository
